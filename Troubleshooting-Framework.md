# IT Troubleshooting Framework

This document outlines the standard troubleshooting methodology used in IT support environments. It follows the CompTIA methodology.

## Step 1 – Identify the Problem

Gather user information and understand the issue.

Actions:

* Ask the user questions 
* Identify symptoms 
* Check error messages
* Determine scope of the issue
* Identify recent changes

Example:
User reports printer shows "Offline" and print jobs remain in queue.

---

## Step 2 – Establish a Theory of Probable Cause

Determine possible causes starting with the simplest.
Assume what the most likely cause is.

Example causes:
* Incorrect printer selected
* Stuck print queue
* Printer spooler service stopped
* Network connectivity issue

---

## Step 3 – Test the Theory to Determine Cause

Run tests to confirm or reject the theory.

Example tests:

* Ping device IP
* Restart printer spooler service
* Check network connectivity
* Review configuration settings
* Remove and re-add the printer

If the test fails, return to Step 2 and test another theory.

---

## Step 4 – Establish a Plan of Action to Resolve the Problem and Implement the Solution

Determine the best solution and apply the fix.

Example:
Remove incorrect printer mapping and reinstall network printer.

---

## Step 5 – Verify Full System Functionality and Implement Preventative Measures

Confirm the issue is resolved and the system functions normally.

Example:
Send a test print and verify the document prints successfully.

---

## Step 6 – Document Findings, Actions, and Outcomes

Record the troubleshooting process.

Documentation should include:

* Problem description
* Root cause
* Steps taken
* Resolution
* Final system status
